# 小舆 - 舆情与品控数据分析助手 部署文档

## 整体架构

```
用户在飞书 @小舆 机器人 → 飞书事件回调 → 飞书云函数 → DeepSeek API + 飞书多维表格 API → 消息卡片回复
```

---

## 一、创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn) → 点击"创建应用" → "自建应用"
2. 填写应用名称：**小舆**，描述：舆情与品控数据分析助手
3. 创建完成后进入应用配置页

### 1.1 启用机器人

左侧菜单 → "应用功能" → "机器人" → 开启"启用机器人"

### 1.2 配置权限

左侧菜单 → "权限管理" → 添加以下权限：

| 权限 | 代码 | 说明 |
|------|------|------|
| 获取多维表格数据 | `bitable:app` | 查询舆情登记表 |
| 获取与发送单聊消息 | `im:message` | 接收和回复消息 |
| 以应用身份发送消息 | `im:message:send_as_bot` | 机器人发消息 |
| 获取用户 Open ID | `contact:user.employee_id:readonly` | 识别用户 |

### 1.3 配置事件订阅

左侧菜单 → "事件与回调" → "事件配置" → 添加事件：

- `im.message.receive_v1` — 接收用户消息

订阅方式选择 **"回调路由"**，URL 填写云函数的 HTTPS 触发地址（部署云函数后会生成）。

### 1.4 获取凭证

左侧菜单 → "凭证与基础信息" → 记录 **App ID** 和 **App Secret**

## 二、将应用添加为舆情登记表协作者

三个舆情登记表都需要添加：

1. 打开舆情登记表 → 右上角"..." → "更多" → "添加文档管理者"
2. 搜索应用名称"小舆" → 添加为 **编辑** 权限
3. 对以下三个 Base 都执行此操作：
   - 2024年舆情登记表
   - 2025年舆情登记表
   - 【2026】舆情登记表

## 三、部署飞书云函数

### 3.1 在飞书开放平台创建云函数

1. 左侧菜单 → "云开发" → "云函数"
2. 点击"创建云函数" → 名称：`xiaoyu-agent`
3. 运行时选择：**NodeJS 20**
4. 内存：**512MB**
5. 超时时间：**120秒**（DeepSeek 生成需要时间）

### 3.2 上传代码

有两种方式：

**方式 A：直接上传 ZIP**
```bash
# 在项目目录下打包
cd xiaoyu-function
npm install
# 打包 node_modules + 所有 .js 文件
zip -r xiaoyu.zip . -x "*.git*"
```
然后在飞书云函数页面上传 ZIP。

**方式 B：连接 Git 仓库**
在云函数配置中绑定 Git 仓库，自动同步代码。

### 3.3 配置环境变量

在云函数配置中添加以下环境变量：

| 变量名 | 值 |
|--------|-----|
| `FEISHU_APP_ID` | 你的飞书应用 App ID |
| `FEISHU_APP_SECRET` | 你的飞书应用 App Secret |
| `DEEPSEEK_API_KEY` | 你的 DeepSeek API Key |
| `DEEPSEEK_MODEL` | `deepseek-v4-flash`（默认，也可用 pro） |

### 3.4 获取云函数触发地址

部署完成后，在云函数详情页找到 **HTTPS 触发地址**（格式如 `https://<function-id>.feishu.cn/...`）。

### 3.5 配置事件回调 URL

回到"事件与回调" → 把刚才的 HTTPS 触发地址填入"回调路由"URL 中。

## 七、常见问题

**Q：云函数超时怎么办？**
A：DeepSeek 调用耗时较长，已在 spec.json 中设置 120 秒超时。如果仍超时，可以尝试将 DEEPSEEK_MODEL 改为 `deepseek-v4-flash`（速度更快）。

**Q：查不到数据**
A：确认应用已添加为三个 Base 的协作者。确认环境变量中的 FEISHU_APP_ID 和 FEISHU_APP_SECRET 正确。

**Q：提示权限不足**
A：检查权限管理中是否已添加 `bitable:app`、`im:message`、`im:message:send_as_bot`，并且应用已发布。
